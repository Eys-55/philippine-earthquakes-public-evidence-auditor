## Current Premise

The tracker primitives exist, but the Matt Pocock front door is not enforced.
The router must immediately create or lock a workflow run when the user reports
a bug, then run a `workflow_intake` phase that loads context before diagnosis,
PRD creation, issue slicing, or implementation.

## Decisions So Far

- Bug reports immediately create or lock a tracked workflow run.
- The first phase is `workflow_intake`.
- `workflow_intake` records the raw user report and rough generated title.
- Context loading happens inside the tracked workflow run.
- A first-class context manifest is required before diagnosis.
- The manifest lives beside workflow-run logs under `ops/workflow-runs/`.
- Routine GitHub upload for this repo is pre-approved as part of workflow
